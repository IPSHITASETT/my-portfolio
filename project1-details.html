Wallview - Multi-Platform Media Streaming Solution

dev.v2.wallview.net

A production-grade multimedia platform that transforms how organizations and families share digital content. Built entirely from scratch with a small frontend team, Wallview delivers features including password-protected streaming, cross-device synchronization, and integrated payment processing for subscription management.

Project Context: Developed as a from-scratch project where our frontend team (2-3 developers) built the complete user interface based on Figma mockups. We worked closely with the backend team to integrate APIs and implement all features.

  • Implemented payment processing with success/failure handling and user notifications

Challenge 1: Learning and Integrating Square Payment SDK

Problem: Had to integrate Square Payment SDK for the first time without prior payment gateway experience. Needed to handle various payment scenarios like successful payments, failed payments, and card validation.

Solution: Studied Square's documentation thoroughly and implemented the SDK step by step. Created a test environment using Square's sandbox mode to test all scenarios before going live. Built comprehensive error messages to guide users when payments fail.

Challenge 2: Real-time Updates Across Multiple Devices

Problem: When users update playlists on their phone, the changes need to show immediately on connected TVs without manual refresh.

Solution: Implemented instant UI updates (optimistic updates) so users see changes immediately. Used Redux to manage state and automatically refetch data when changes occur. Set up periodic checking for updates on connected devices.

Challenge 3: Secure Cookie-based Authentication

Problem: Needed to implement secure authentication where login tokens are stored safely and can't be accessed by malicious scripts.

Solution: Used HTTP-only cookies (cookies that can't be accessed by JavaScript) instead of localStorage. This prevents hackers from stealing user sessions. Implemented automatic token refresh so users stay logged in seamlessly.

Challenge 4: Uploading Large Video Files

Problem: Users needed to upload large video files (sometimes several GBs) without timeouts or failures.

Solution: Implemented chunked uploads where large files are broken into smaller pieces and uploaded one by one. Added progress bars showing upload percentage and estimated time remaining. If upload fails, users can resume from where it stopped.

Challenge 5: Making UI Work on TV Screens

Problem: Designing interface that works well on both small phone screens and large TV screens with remote control navigation.

Solution: Started with mobile design first, then enhanced for larger screens. Made buttons bigger and navigation simpler for TV. Removed hover effects since TVs don't have mouse. Used CSS Grid and Flexbox for layouts that adapt to any screen size.
