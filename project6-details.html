FreshKrate - Fresh Produce Delivery Mobile App

freshkrate.com

A cross-platform mobile application for fresh produce delivery services, connecting customers with local farmers and fresh food suppliers. Built using React Native and TypeScript for iOS and Android platforms, with Firebase as the backend infrastructure. Worked in a small team of 2-3 developers to implement key features and functionality.

Challenge 1: TypeScript Learning Curve

Problem: First time working with TypeScript in a React Native project. Had to understand interfaces, types, generics, and type annotations while developing features.

Solution: Spent time learning TypeScript basics and best practices. Started with simple types for component props and state, then gradually implemented more complex types. Used TypeScript documentation and team code reviews to improve type safety. This resulted in fewer runtime bugs and better code maintainability.

Challenge 2: Cross-Platform UI Consistency

Problem: Ensuring UI components look and behave consistently on both iOS and Android devices, as each platform has different design guidelines and behaviors.

Solution: Tested every feature on both iOS simulator and Android emulator regularly. Used Platform API from React Native to apply platform-specific styles when needed. Created reusable styled components that adapt to platform automatically. Fixed padding, margin, and font rendering differences between platforms.

Challenge 3: Real-time Cart Updates with Redux

Problem: Managing shopping cart state across multiple screens while keeping data synchronized with Firebase backend.

Solution: Implemented Redux for centralized cart state management. Created actions for add/remove/update cart items. Connected components using useSelector and useDispatch hooks. Synced cart data with Firebase Firestore so cart persists across app sessions and devices. Added optimistic updates for instant UI feedback.

Challenge 4: Firebase Firestore Query Optimization

Problem: Initial implementation fetched all products at once, causing slow loading times and high data usage.

Solution: Implemented pagination using Firestore's limit() and startAfter() methods. Added category-based filtering to fetch only relevant products. Implemented caching strategy to avoid repeated queries. Used Firestore indexes for faster complex queries. This reduced initial load time by 60%.

Challenge 5: Handling Async Operations and Loading States

Problem: Managing multiple asynchronous Firebase calls (authentication, data fetching) and showing appropriate loading states to users.

Solution: Used async/await with try-catch blocks for error handling. Implemented loading indicators using state variables. Created reusable Loading component shown during data fetching. Added error messages with retry options when operations fail. Used TypeScript to ensure proper error typing.

Challenge 6: Navigation State Management

Problem: Complex navigation flows between product browsing, cart, checkout, and order tracking needed to maintain proper navigation stack.

Solution: Structured navigation using Stack Navigator for main flows and Tab Navigator for bottom navigation. Implemented deep linking for notifications to specific screens. Used navigation params to pass data between screens efficiently. Reset navigation stack after checkout completion to prevent going back to cart.

Challenge 7: Image Loading Performance

Problem: Product images from Firebase Storage were loading slowly, especially on slower networks, affecting user experience.

Solution: Implemented React Native Fast Image library for better image caching and loading. Added placeholder images while actual images load. Compressed and resized images in Firebase Storage. Implemented lazy loading for product lists - images load only when visible on screen. This improved perceived performance significantly.

Challenge 8: Team Code Conflicts and Collaboration

Problem: Working with 2-3 developers on same codebase sometimes led to merge conflicts and integration issues.

Solution: Established clear branching strategy (feature branches, develop, main). Communicated regularly about which files/features we're working on. Performed code reviews before merging to catch issues early. Used consistent code formatting with Prettier and ESLint. Broke features into smaller tasks to minimize conflicts.
