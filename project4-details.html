Challenge 1: Managing 250+ Products with Real-time Inventory

Problem: Needed to handle large product catalog with real-time stock updates across website and mobile apps. When stock changes in admin dashboard, it should instantly reflect everywhere.

Solution: Used Firebase Realtime Database's live synchronization feature. Optimized database structure with indexed queries for fast product searches. Implemented caching strategy to reduce database reads and improve performance.

Challenge 2: Building UI Without Figma Designs

Problem: Product-based project with no pre-made designs. Had to create UI directly based on internal discussions and client requirements.

Solution: Leveraged Material UI component library for consistent, professional design. Created design system with reusable components. Iterated quickly based on client feedback in weekly demos. Built flexible layouts that could adapt as requirements evolved.

Challenge 3: Coordinating Website with Mobile Apps

Problem: Website, Android app, and iOS app needed to share same data and stay synchronized. Changes in one platform should reflect in others instantly.

Solution: Used Firebase as single source of truth for all platforms. Created standardized API structure that all platforms consume. Built admin dashboard controlling all three platforms from one place. Established weekly sync meetings with app developers to align features.

Challenge 4: Handling Complex State Management

Problem: Managing cart items, user data, product filters, and order history across multiple pages while keeping UI responsive.

Solution: Combined Zustand for global state (cart, user session) with Context API for feature-specific state (product filters). This hybrid approach kept code organized and performant. Used local storage to persist cart data between sessions.

Challenge 5: Product Variations and Pricing Logic

Problem: Same product (e.g., Chicken) available in different variations (500g, 1kg, whole, cut) with different prices. Admin needed flexibility to configure pricing rules and discounts.

Solution: Designed flexible database schema supporting unlimited variations per product. Built variation selector UI showing all options clearly. Created business logic engine in admin panel where client can set pricing rules, bulk discounts, and seasonal offers without code changes.

Challenge 6: Weekly Updates Without Breaking Production

Problem: Project required weekly deployments with new features while keeping site stable for existing users.

Solution: Implemented feature flagging system to test new features in production safely. Created staging environment mirroring production for thorough testing. Used Firebase's gradual rollout feature to release updates to small user groups first. Maintained comprehensive testing checklist covering all critical user flows.

Challenge 7: Budget Management Across Multiple Deliverables

Problem: Single budget covering website, admin dashboard, and two mobile apps. Needed to optimize costs while delivering quality.

Solution: Chose Firebase for cost-effective scalability (pay only for usage). Reused components across website and admin panel to reduce development time. Negotiated fixed-price contract with app developers with clear deliverables. Tracked expenses weekly and adjusted scope where needed to stay within budget.
